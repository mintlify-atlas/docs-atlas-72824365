---
title: Quickstart
description: Run StatHub locally in a few minutes by setting up the server and client side by side.
---

## Prerequisites

- **Node.js** v18 or later
- **pnpm** — install with `npm install -g pnpm`
- **PostgreSQL** database (local or remote)

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <your-repo-url>
    cd stathub
    ```
  </Step>

  <Step title="Install server dependencies">
    ```bash
    cd server
    pnpm install
    ```
  </Step>

  <Step title="Configure server environment variables">
    Create a `.env` file inside the `server/` directory. Use `.env.example` as the template:

    ```env
    PORT=8000
    HOST=0.0.0.0
    DATABASE_URL=postgres://user:password@localhost:5432/stathub

    # Arcjet
    ARCJET_KEY=
    ARCJET_ENV=development
    ARCJET_MODE=

    # Auth
    JWT_SECRET=
    GOOGLE_CLIENT_ID=
    GOOGLE_CLIENT_SECRET=

    # Email
    RESEND_API_KEY=

    # Client
    CLIENT_URL=http://localhost:5173
    CLIENT_PASSWORD_RESET_URL=http://localhost:5173/reset-password
    ```

    | Variable | Description |
    |---|---|
    | `PORT` | Port the server listens on. Defaults to `8000`. |
    | `HOST` | Bind address. Defaults to `0.0.0.0`. |
    | `DATABASE_URL` | PostgreSQL connection string. |
    | `ARCJET_KEY` | Arcjet API key from your Arcjet dashboard. |
    | `ARCJET_ENV` | Set to `development` locally. |
    | `JWT_SECRET` | Secret used to sign JWTs. Use a long random string. |
    | `GOOGLE_CLIENT_ID` | OAuth 2.0 client ID from Google Cloud Console. |
    | `GOOGLE_CLIENT_SECRET` | OAuth 2.0 client secret from Google Cloud Console. |
    | `RESEND_API_KEY` | API key for sending transactional email via Resend. |
    | `CLIENT_URL` | Origin of the client app, used for CORS. |
    | `CLIENT_PASSWORD_RESET_URL` | Full URL to the client password-reset page. |
  </Step>

  <Step title="Run database migrations">
    Generate and apply the Drizzle schema to your PostgreSQL database:

    ```bash
    pnpm db:generate
    pnpm db:migrate
    ```

    <Tip>
      Run `pnpm db:studio` to open Drizzle Studio and inspect your database in the browser.
    </Tip>
  </Step>

  <Step title="Start the server">
    ```bash
    pnpm dev
    ```

    The server starts with Node's built-in `--watch` flag so it restarts automatically on file changes.

    You should see:

    ```
    Server is running at http://localhost:8000
    WebSocket server is running at ws://localhost:8000/ws
    ```
  </Step>

  <Step title="Install client dependencies">
    Open a new terminal from the repo root:

    ```bash
    cd client
    pnpm install
    ```
  </Step>

  <Step title="Configure client environment variables">
    Create a `.env` file inside the `client/` directory:

    ```env
    VITE_API_BASE_URL=http://localhost:8000
    VITE_GOOGLE_CLIENT_ID=
    ```

    | Variable | Description |
    |---|---|
    | `VITE_API_BASE_URL` | Base URL of the StatHub API server. |
    | `VITE_GOOGLE_CLIENT_ID` | Same Google OAuth client ID used by the server. |
  </Step>

  <Step title="Start the client">
    ```bash
    pnpm dev
    ```

    Vite starts the development server. Open [http://localhost:5173](http://localhost:5173) in your browser.
  </Step>

  <Step title="Verify the setup">
    Check the health endpoint:

    ```bash
    curl http://localhost:8000/
    # Server is up and running!
    ```

    Test the WebSocket endpoint by connecting to `ws://localhost:8000/ws`. Any client that connects will receive `match_created` and commentary events as they are broadcast by the server.
  </Step>
</Steps>

## Available server scripts

| Script | Command | Description |
|---|---|---|
| Start (production) | `pnpm start` | Runs `node src/index.js` directly. |
| Start (development) | `pnpm dev` | Runs with `--watch` for auto-restart. |
| Generate migrations | `pnpm db:generate` | Generates SQL migration files from the Drizzle schema. |
| Apply migrations | `pnpm db:migrate` | Applies pending migrations to the database. |
| Open DB studio | `pnpm db:studio` | Launches Drizzle Studio in the browser. |

## Available client scripts

| Script | Command | Description |
|---|---|---|
| Development | `pnpm dev` | Starts the Vite dev server with HMR. |
| Build | `pnpm build` | Produces an optimized production build in `dist/`. |
| Preview | `pnpm preview` | Serves the production build locally for testing. |
