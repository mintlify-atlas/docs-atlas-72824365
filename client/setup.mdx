The StatHub client is a React 19 application built with Vite, styled with Tailwind CSS v4, and communicates with the StatHub server over HTTP.

## Prerequisites

- Node.js 18 or later
- [pnpm](https://pnpm.io) (the project uses pnpm as its package manager)
- A running StatHub server (the client depends on `VITE_API_BASE_URL` pointing to it)
- A Google OAuth client ID if you want Google sign-in to work

## Setup

<Steps>
  <Step title="Clone the repository and navigate to the client directory">
    ```bash
    cd client
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example env file and fill in your values:

    ```bash
    cp .env.example .env
    ```

    Then edit `.env`:

    ```env
    VITE_GOOGLE_CLIENT_ID=your-google-oauth-client-id
    VITE_API_BASE_URL=http://localhost:3000
    ```

    | Variable | Description |
    |---|---|
    | `VITE_GOOGLE_CLIENT_ID` | Google OAuth 2.0 client ID for the "Continue with Google" button on login and signup pages |
    | `VITE_API_BASE_URL` | Base URL of the StatHub server API, used for all authentication requests |

    <Note>
      All Vite environment variables must be prefixed with `VITE_` to be accessible in the browser bundle.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    pnpm dev
    ```

    Vite starts a local dev server, typically at `http://localhost:5173`, with hot module replacement enabled.
  </Step>
</Steps>

## Build for production

```bash
pnpm build
```

Vite compiles and bundles the application into the `dist/` directory. The build uses the `babel-plugin-react-compiler` plugin for optimized React output.

## Preview the production build

```bash
pnpm preview
```

Serves the contents of `dist/` locally so you can verify the production build before deploying.

<Tip>
  Run `pnpm build && pnpm preview` together to catch any issues that only appear in the production bundle.
</Tip>

## How the client connects to the server

All API calls are made through an Axios instance configured in `src/api/auth.js`. The instance uses `VITE_API_BASE_URL` as its `baseURL`:

```js
const authApi = axios.create({
  baseURL: CONFIG.API_BASE_URL,
  headers: {
    "Content-Type": "application/json",
  },
});
```

The `CONFIG` object in `src/config/config.js` reads the Vite env vars at build time:

```js
export const CONFIG = {
  GOOGLE_CLIENT_ID: import.meta.env.VITE_GOOGLE_CLIENT_ID,
  API_BASE_URL: import.meta.env.VITE_API_BASE_URL,
  ENDPOINTS: {
    AUTH: {
      LOGIN: "/api/auth/login",
      SIGNUP: "/api/auth/signup",
      GOOGLE: "/api/auth/google",
      FORGOT_PASSWORD: "/api/auth/forgot-password",
      RESET_PASSWORD: "/api/auth/reset-password",
    },
  },
};
```

<Warning>
  If `VITE_API_BASE_URL` is not set, all API requests will fail. Make sure the StatHub server is running and the URL is reachable from the browser before starting the client.
</Warning>
