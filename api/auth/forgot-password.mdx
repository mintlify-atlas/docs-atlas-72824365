---
title: Forgot password
description: Request a password reset email containing a one-time token link.
---

`POST /api/auth/forgot-password`

Generates a UUID reset token, stores it in the user's `metadata` field, and sends an email via Resend containing a link to the client reset page. The token expires after **1 hour**.

<Note>
  This endpoint always returns `200 OK` regardless of whether the email address belongs to a registered account. This prevents user enumeration.
</Note>

## How it works

1. The server looks up a user by email in the database.
2. If no user is found, it returns a success response immediately without sending an email.
3. If a user is found, the server generates a `uuid v4` token and sets `user.metadata.resetPasswordToken` and `user.metadata.resetPasswordExpires` (1 hour from now).
4. An email is sent from `auth@stathub.yeeshu.dev` via Resend with a link constructed from the `CLIENT_PASSWORD_RESET_URL` environment variable and the token as a query parameter.

## Request body

<ParamField body="email" type="string" required>
  The email address of the account to reset. Must be a non-empty string.
</ParamField>

## Response

**200 OK**

```json
{ "message": "Password reset instructions sent to your email" }
```

This message is returned whether or not the email address is registered.

## Error responses

| Status | Error | Cause |
|--------|-------|-------|
| `400` | `Email is required` | The `email` field was missing from the request body |
| `500` | `Failed to send reset email` | The Resend API returned an error |
| `500` | `Something went wrong` | Unexpected server error |

## Example

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:8000/api/auth/forgot-password \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "yeeshu@example.com"
  }'
```

</CodeGroup>

```json Response
{
  "message": "Password reset instructions sent to your email"
}
```
