---
title: Reset password
description: Set a new password using the token received in the password reset email.
---

`POST /api/auth/reset-password`

Looks up the user by their reset token, checks that the token has not expired, hashes the new password with bcrypt, saves it, and removes the reset token from the user's metadata.

## How it works

1. The server queries for a user whose `metadata.resetPasswordToken` matches the supplied `token`.
2. If no user is found, it returns `400 Invalid or missing token`.
3. If `metadata.resetPasswordExpires` is in the past, it returns `400 Token has expired`.
4. The new password is hashed with bcrypt (12 salt rounds) and saved.
5. `resetPasswordToken` and `resetPasswordExpires` are deleted from the user's metadata.

## Request body

<ParamField body="token" type="string" required>
  The UUID reset token from the password reset email link.
</ParamField>

<ParamField body="newPassword" type="string" required>
  The new plain-text password to set. Stored as a bcrypt hash.
</ParamField>

## Response

**200 OK**

```json
{ "message": "Password has been successfully reset" }
```

## Error responses

| Status | Error | Cause |
|--------|-------|-------|
| `400` | `Token and new password are required` | `token` or `newPassword` was missing from the request body |
| `400` | `Invalid or missing token` | No user record has a matching `resetPasswordToken` in their metadata |
| `400` | `Token has expired` | The token exists but `resetPasswordExpires` is in the past |
| `500` | `Something went wrong` | Unexpected server error |

## Example

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:8000/api/auth/reset-password \
  --header 'Content-Type: application/json' \
  --data '{
    "token": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "newPassword": "NewPass5678"
  }'
```

</CodeGroup>

```json Response
{
  "message": "Password has been successfully reset"
}
```
