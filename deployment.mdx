---
title: Deployment
description: Deploy StatHub's server and client as Docker containers using the included Dockerfiles and deploy scripts.
---

Both the server and the client ship with multi-stage Dockerfiles. The server image runs the Node.js process directly; the client image builds the Vite app and serves it with nginx.

<Warning>
  Never commit `.env` files or secret values to source control. In production, supply environment variables through your container runtime, secrets manager, or CI/CD pipeline rather than from a file on disk.
</Warning>

<Tabs>
  <Tab title="Server">
    ## Server Dockerfile

    The server uses a two-stage build. The builder stage installs dependencies, and the production stage copies the result and runs the app.

    ```dockerfile server/Dockerfile
    FROM node:24.14.0-alpine AS builder

    RUN npm i -g pnpm

    WORKDIR /app

    COPY package.json pnpm-lock.yaml ./
    COPY src ./src

    RUN pnpm install --frozen-lockfile

    FROM node:24.14.0-alpine AS production

    WORKDIR /app

    COPY --from=builder /app/ /app

    EXPOSE 8000

    CMD ["node", "src/index.js"]
    ```

    ### Build and run manually

    ```bash
    # Build the image
    docker build -t stathub-server ./server

    # Run the container
    docker run -d \
      -p 8000:8000 \
      --name stathub-server \
      --env-file server/.env \
      stathub-server
    ```

    ### Production environment variables

    All variables from `server/.env.example` must be set at runtime:

    ```env
    PORT=8000
    HOST=0.0.0.0
    DATABASE_URL=postgres://user:password@host:5432/stathub
    ARCJET_KEY=
    ARCJET_ENV=production
    ARCJET_MODE=
    JWT_SECRET=
    GOOGLE_CLIENT_ID=
    GOOGLE_CLIENT_SECRET=
    RESEND_API_KEY=
    CLIENT_URL=https://your-client-domain.com
    CLIENT_PASSWORD_RESET_URL=https://your-client-domain.com/reset-password
    ```

    ### Using the deploy script

    `server/deploy_server.sh` pulls a pre-built image from Docker Hub and (re)starts the container. Set these variables before running it:

    | Variable | Description |
    |---|---|
    | `DOCKER_USERNAME` | Docker Hub username. |
    | `DOCKER_PASSWORD` | Docker Hub password or access token. |
    | `IMAGE_NAME` | Full image name, e.g. `myuser/stathub-server:latest`. |
    | `CONTAINER_NAME` | Name to assign the running container. |
    | `PORT_OUTSIDE` | Host port to expose, e.g. `8000`. |
    | `PORT_INSIDE` | Container port to map to, e.g. `8000`. |

    The script expects a `.server.env` file in the working directory that is passed to the container via `--env-file`.

    ```bash
    export DOCKER_USERNAME=myuser
    export DOCKER_PASSWORD=mytoken
    export IMAGE_NAME=myuser/stathub-server:latest
    export CONTAINER_NAME=stathub-server
    export PORT_OUTSIDE=8000
    export PORT_INSIDE=8000

    bash server/deploy_server.sh
    ```

    <Note>
      The script installs Docker automatically if it is not already present on the host (Debian/Ubuntu only, via `apt-get`).
    </Note>

    ## Client Dockerfile

    The client uses a two-stage build. The build stage compiles the Vite app; the second stage serves the static output with nginx 1.28.

    ```dockerfile client/Dockerfile
    # Stage 1: Build step
    FROM node:24.14.0-alpine AS build-step

    WORKDIR /usr/src/app

    COPY package.json ./
    COPY pnpm-lock.yaml ./

    RUN npm install -g pnpm && pnpm install --frozen-lockfile

    COPY . .

    ARG VITE_API_BASE_URL
    ARG VITE_GOOGLE_CLIENT_ID

    RUN pnpm run build

    # Stage 2: Nginx server
    FROM nginx:1.28.2-alpine

    COPY nginx.conf /etc/nginx/nginx.conf

    COPY --from=build-step /usr/src/app/dist /usr/share/nginx/html

    CMD ["nginx", "-g", "daemon off;"]
    ```

    The `VITE_API_BASE_URL` and `VITE_GOOGLE_CLIENT_ID` values are baked into the bundle at build time via Docker build arguments. Pass them with `--build-arg`.

    ### nginx configuration

    The client image ships a custom nginx config that enables HTML5 history-mode routing:

    ```nginx client/nginx.conf
    user nginx;
    worker_processes auto;

    error_log /var/log/nginx/error.log warn;
    pid /var/run/nginx.pid;

    events {
        worker_connections 1024;
    }

    http {
        include /etc/nginx/mime.types;
        default_type application/octet-stream;

        access_log /var/log/nginx/access.log main;
        sendfile on;
        keepalive_timeout 65;

        server {
            listen 80;
            location / {
                root /usr/share/nginx/html;
                index index.html index.htm;
                try_files $uri $uri/ /index.html;
            }
        }
    }
    ```

    The `try_files` directive ensures React Router can handle client-side navigation without returning 404 on hard refresh.

    ### Build and run manually

    ```bash
    # Build the image (supply Vite env vars as build args)
    docker build \
      --build-arg VITE_API_BASE_URL=https://your-api-domain.com \
      --build-arg VITE_GOOGLE_CLIENT_ID=your-google-client-id \
      -t stathub-client ./client

    # Run the container
    docker run -d \
      -p 80:80 \
      --name stathub-client \
      stathub-client
    ```

    ### Using the deploy script

    `client/deploy_client.sh` pulls a pre-built image from Docker Hub and (re)starts the container. Set these variables before running it:

    | Variable | Description |
    |---|---|
    | `DOCKER_USERNAME` | Docker Hub username. |
    | `DOCKER_PASSWORD` | Docker Hub password or access token. |
    | `IMAGE_NAME` | Full image name, e.g. `myuser/stathub-client:latest`. |
    | `CONTAINER_NAME` | Name to assign the running container. |
    | `PORT_OUTSIDE` | Host port to expose, e.g. `80`. |
    | `PORT_INSIDE` | Container port to map to, e.g. `80`. |

    ```bash
    export DOCKER_USERNAME=myuser
    export DOCKER_PASSWORD=mytoken
    export IMAGE_NAME=myuser/stathub-client:latest
    export CONTAINER_NAME=stathub-client
    export PORT_OUTSIDE=80
    export PORT_INSIDE=80

    bash client/deploy_client.sh
    ```

    <Note>
      Unlike the server script, the client deploy script does not pass an `--env-file` to the container because all client configuration is compiled into the static bundle at build time.
    </Note>
